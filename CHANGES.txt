WHAT WAS FIXED AND CHANGED
===========================

1. DUPLICATE DETECTION BUG (AddTransaction.tsx)
   - Old: bulk-add only checked against existing DB transactions (stale snapshot)
   - Fix: now tracks items added within the same batch so duplicates in one Excel
     file are also caught and skipped
   - UI also shows duplicate warnings for same-batch duplicates now

2. USE TOAST LISTENER LEAK (hooks/use-toast.ts)
   - Old: useEffect had [state] as dependency, causing listener to re-register
     on every state change = memory leak
   - Fix: changed dependency to [] (empty)

3. DATE.NOW() AS REACT KEY (AddTransaction.tsx)
   - Old: key={Date.now()} on TransactionForm forced full remount every render
   - Fix: removed, since scannedData prop already controls form reset via effect

4. DUAL TOAST SYSTEMS (App.tsx)
   - Old: both Radix <Toaster /> and Sonner <Sonner /> were rendered
   - Fix: removed Radix Toaster, kept only Sonner (all toast() calls use Sonner)

5. ARRAY INDEX AS REACT KEY (AddTransaction.tsx)
   - Old: key={idx} (array index) in scanned results table
   - Fix: key={`${note}-${amount}-${date}-${idx}`} (stable composite key)

6. DEPRECATED readAsBinaryString (AddTransaction.tsx)
   - Old: reader.readAsBinaryString(file) - deprecated File API
   - Fix: reader.readAsArrayBuffer(file) with XLSX.read(type: "array")

7. EMPTY CATCH BLOCK (lib/ocr.ts)
   - Old: catch(e) {} - silently swallowed date parsing errors
   - Fix: added console.warn to log the error

8. BULK DELETE ABORTS ON FIRST FAILURE (History.tsx)
   - Old: for loop with await - if one delete failed, remaining were skipped
   - Fix: Promise.allSettled with failure count toast

9. COMPETING USEEFFECT HOOKS (TransactionForm.tsx)
   - Old: two separate useEffect both calling form.reset() = race condition
   - Fix: merged into one effect with if/else

10. USEISMOBILE INITIAL UNDEFINED (hooks/use-mobile.tsx)
    - Old: useState<boolean | undefined>(undefined)
    - Fix: useState(false)

11. UNUSED IMPORTS REMOVED
    - AddTransaction.tsx: Loader2, FileImage, scanReceipt
    - History.tsx: CheckSquare, Square
    - useTransactions.ts: orderBy, Timestamp

12. CATEGORY DETECTION FOR EXCEL (AddTransaction.tsx + lib/ocr.ts)
    - Old: Excel imports had no category (defaulted to "Other")
    - Fix: now runs detectCategory() on the note/person field
      matching keywords against the payee/note text:

      Food:      restaurant, hotel, cafe, food, burger, pizza, coffee, lunch,
                 dinner, grocery, groceries, mart, swiggy, zomato, vending,
                 bakery, bakers, sweet, sweets, tea, stall, dry fruit, pan,
                 wok, kitchen, choco, kirana
      Rent:      rent, lease, apartment
      Travel:    uber, ola, petrol, fuel, gas, taxi, train, flight, indigo,
                 air, irctc, apsrtc, rtc, bus, airline, railway, rapido,
                 metro, oyo, road transport, parking
      Healthcare: pharmacy, medical, hospital, doctor, clinic, medplus, apollo
      Entertain: movie, cinema, theatre, netflix, spotify, game, bookmyshow
      Shopping:  amazon, flipkart, myntra, ajio, nykaa, meesho, shopping,
                 mall, store, marketings, xerox, zudio, ikea, haier, salon,
                 traders, trading
      Utilities: electricity, water, bill, recharge, mobile, internet, jio,
                 airtel, vi, bsnl, eazypay, fiber, fibre, networks, postpaid
      Savings:   investment, mutual fund, sip, stocks, zerodha, upstox,
                 groww, deposit

    - Income type transactions → category "Income"
    - Person names that don't match any keyword → "Other"

13. CSV vs XLSX PARSING (AddTransaction.tsx)
    - Old: readAsBinaryString + type: "binary" for all files (deprecated, unreliable)
    - Fix: CSV files → readAsText + type: "string"
          XLSX/XLS files → readAsArrayBuffer + Uint8Array + type: "array"

14. CATEGORY COLUMN IN CSV (AddTransaction.tsx + CSV files)
    - New: parser looks for a "Category" column in the sheet
    - If present and non-empty, uses it directly; otherwise falls back to detectCategory()
    - Sample CSVs updated with Category column header
    - june-statement.csv has pre-filled categories for all 50 rows

15. SAMPLE CSV FILES ADDED
    - excel/april-statement.csv (141 transactions)
    - excel/may-statement.csv (70 transactions)
    - excel/june-statement.csv (50 transactions)
    - Columns: Date, Time, Year, Person, Amount (INR), Type (Debit/Credit), Transaction ID, Category
